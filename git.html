Git



  • Git Features
  • It is very fast and safe.
  • Resource stores take up very little space on the disk.
  • Branching and merging operations are easy and fast.
  • Available offline.
  • Since it has a distributed structure, the project can be kept in the machine of each encoder with its entire history.
  • At first it may be thought that this slows down operations and is a waste of disk. However, these problems are never experienced thanks to Git's use of highly effective algorithms.
  • It has a free plan. (public warehouse)
  • Their reports are extensive. All changes made by the users included in the project can be checked from line to line web interface.
  • There are e-mail notifications. The changes are sent to your e-mail address instantly.


Some Basic Git Commands



  • it init

Usage: git init [repository name]

This command is used to start a new repository.



  • git clone

Usage: git clone [url]

This command adds a file to the staging area.



  • git add

Usage: git add [file]

This command is used to start a new repository.



  • git commit

Usage: git commit -m “[ Type in the commit message]”

This command records or snapshots the file permanently in the version history.
